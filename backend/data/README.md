# CanopyOS Data Sources & Intelligence Pipeline

This directory documents the data ecosystem that powers the CanopyOS backend. Our pipeline is designed to unify fragmented Canadian wildfire telemetry, fusing multi-sensor satellite data with high-resolution meteorological models to feed our predictive and tactical RL engines.

## 1. Federal/National Data (The Baseline)
* [cite_start]**CWFIS (Canadian Wildland Fire Information System):** Essential for pulling the Canadian Forest Fire Danger Rating System (CFFDRS) indices[cite: 543]. [cite_start]Crucially, we utilize the national raster of Fire Behaviour Prediction (FBP) fuel types at ~30m resolution, which is the foundational dataset for understanding how local fuels will dictate fire intensity[cite: 547].
* [cite_start]**CIFFC (Canadian Interagency Forest Fire Centre):** Provides national-level situation reports and data on current resource mobilization, critical for understanding interagency asset availability[cite: 646].
* [cite_start]**WildFireSat (Future Integration):** CanopyOS architecture is future-proofed to ingest daily active wildfire data from the upcoming Canadian space-based constellation[cite: 526, 527].

## 2. Provincial Telemetry (The "Siloed" Data)
* **BC Wildfire Service (BCWS) & Alberta Wildfire APIs:** We ingest ArcGIS REST endpoints to pull live, authoritative provincial fire polygons and incident status updates. This satisfies the platform's need for a standardized, multi-jurisdictional Common Operating Picture (COP).

## 3. The Sensor Fusion Stack (Satellite & Airborne)
* [cite_start]**NASA FIRMS (MODIS/VIIRS):** Provides active fire detections and hotspots with global availability within hours, serving as our primary wide-area thermal detection layer[cite: 519].
* [cite_start]**Geostationary Feeds (GOES East/West):** Utilized for high-frequency (sub-hourly) automated heat-anomaly detection to track rapid perimeter expansion over southern Canada[cite: 521, 522].
* [cite_start]**Airborne IR / Drone Telemetry:** CanopyOS supports high-confidence tactical perimeter ingestion from aircraft-mounted sensors (analogous to the California FIRIS program) to penetrate thick smoke when satellites cannot[cite: 531, 533].

## 4. Meteorological Vectors & Predictive Modeling
* [cite_start]**ECCC HRDPS (High Resolution Deterministic Prediction System):** We pull from Environment and Climate Change Canada to get high-resolution deterministic predictions with 48-hour horizons for wind speed, direction, and terrain data[cite: 541]. This is the primary driver for our XGBoost spread model.
* [cite_start]**Prometheus Compatibility:** While we utilize ML (XGBoost) for rapid 24-hour inference, our data structures remain compatible with outputs from Prometheus, Canada's operational deterministic fire growth simulator[cite: 557, 560].

## 5. Tactical Command & Containment Strategy (RL Engine Inputs)
Our Reinforcement Learning agent requires specific data layers to optimize resource deployment:
* [cite_start]**Potential Control Locations (PCL) & Suppression Difficulty Index (SDI) Proxies:** By combining the 30m FBP fuel rasters with topographical data and road networks, we calculate geographic zones where containment lines are most likely to hold and where it is safest for crews to operate[cite: 583, 584, 587].
* [cite_start]**ICS-Aligned Resource Tasking:** The platform's output recommendations are designed to map to standard Incident Command System (ICS) forms, ensuring interoperability with Canadian pan-jurisdictional workflows[cite: 632, 692].

## 6. Frontend & Visualization
* **Mapbox GL JS:** Powers the dark-mode mission control dashboard. Mapbox renders the base topography, visualizes the XGBoost burn probability grids, and plots the high-contrast glowing "choke point" nodes generated by the Tactical RL engine.
