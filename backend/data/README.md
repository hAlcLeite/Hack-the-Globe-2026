# CanopyOS Data Sources

This folder contains information about the data sets and APIs used to power the CanopyOS intelligence pipeline.

## 1. Federal/National Data
- **CWFIS (Canadian Wildland Fire Information System)**: Essential for pulling the Canadian Forest Fire Danger Rating System (CFFDRS) indices and NRCan's Fire M3 hotspots.
- **CIFFC (Canadian Interagency Forest Fire Centre)**: Great for national-level situation reports and data on current resource mobilization.

## 2. Provincial Telemetry (The "Siloed" Data)
- **BC Wildfire Service (BCWS) API**: They offer excellent ArcGIS REST endpoints (via the BC Data Catalogue) for active fire perimeters, incidents, and weather station data.
- **Alberta Wildfire Open Data**: Similar ArcGIS dashboards and REST services bringing in real-time provincial fire polygons.

## 3. Satellite & Meteorological Vectors
- **NASA FIRMS (Fire Information for Resource Management System) API**: The absolute best source for near real-time MODIS and VIIRS satellite hotspot telemetry. It's fast and easy to parse, perfectly hitting your "Sentinel-2/MODIS" requirement.
- **Environment and Climate Change Canada (ECCC) Datamart**: Critical for fetching the live meteorological vectors (wind speed/direction, temperature, humidity) needed to feed the XGBoost predictive model.

## 4. Tactical Command & Frontend
- **Mapbox APIs**: Since your architecture specifies a Next.js frontend with a "dark-mode Mapbox GL interface," we will definitely need Mapbox GL JS and their APIs to render the base map, draw the predictive fire perimeters, and plot the glowing "choke points" generated by the RL agent.
